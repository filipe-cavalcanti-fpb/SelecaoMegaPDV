Segue um guia para montar a sua documentação:
    
    Link -> Caso adicione links (como tutoriais do Youtube ou alguma outra página WEB), criar um documento (links.txt) somente para os links e abaixo de cada link relatar a sua ultilidade para o projeto;
    
    Artigos -> Caso adicione artigos, criar uma pasta com os artigos (o nome pode ser Artigos), crie um arquivo *.txt e coloque o nome dos artigos mais importantes e uma rápida descrição de sua relevância para o projeto;
    
    Tecnologias -> criar um arquivo (pode ser *.docx ou *.txt) relatanto as tecnologias que se ultilizou e que se está ultilizando até esta fase do projeto EX.: "No começo desenvolvemos uma aplicação Android ultilizando o Android Studio, no entanto optamos por portabilidade e agora desenvolvemos uma aplicação web usando o framework Django[...] a parte do cliente (cliente-side) é feita com JavaScript, o framework Jquery e HTML5 o design da página é estilizado com CSS e Bootstrap, a geraçao dos gráficos ultiliza da API Google Charts[...]", ou seja, relate o uso das tecnologias no decorrer do projeto;
    
    Código Fonte -> coloque todo seu código fonte numa pasta chamada src, claro!!!, dentro da pasta organize seu código o máximo possível, e dentro da pasta coloque um arquivo *.txt ou *.docx contendo uma rápida explicação de seu código. Caso esteja programando baseado em projetos (vários arquivos fazendo parte de uma mesma solução), seja rápido na descrição dos arquivos que fazem parte do seu projeto, e descreva melhor a sua solução (seu projeto);
    
    Demais Documentação -> Caso tenha mais coisas para adicionar e acha que não se encaixam em nemhum dos tópicos acima pode coloca-lá direto na pasta documentation, e a descrição de seu uso/relevância no projeto, coloque no final deste arquivo, onde há uma área destinada para isso. Coloque o nome do arquivo/pasta seguido de sua descrição.
